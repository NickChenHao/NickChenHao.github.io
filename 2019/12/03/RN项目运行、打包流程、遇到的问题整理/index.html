RN项目运行、打包流程、遇到的问题整理

发表于 2019-12-03 | 分类于 aboutJub | |

链接（下文提及的链接）

原生集成code-push热更新RN功能

环境

安装过yarn
安装过cocoapods
注册过code-push账号
项目ios部分 配置好code-push ( 可点击文章头部的链接查看详情 )

运行

  1. svn上check out项目
  2. cd项目根目录 执行yarn命令
  3. 检查根目录是否有relseae_ios文件 若无执行react-native bundle --entry-file index.js --platform ios --dev false --bundle-output relseae_ios/main.jsbundle --assets-dest relseae_ios命令
  4. 将relseae_ios中的main.jsbundle和asset 拖拽到ios项目中 拖拽到ios项目根目录中 （copy选择真实文件夹 蓝色的）
  5. cd ios项目目录 执行pod install命令
  6. xcode command+R 运行

运行中可能遇到的问题

  • pod  OpenSSL 失败问题

  • 有时你用Xcode跑工程时 可能会出现这个问题 Script-00DD1BFF1BD5951E006B06BC.sh: line 3: ../node_modules/react-native/packager/react-native-xcode.sh: No such file or directory

  • Faild to load bundle (http://localites:8081/index.bundle?platform=ios$dev=true)···
  • Unable to resolve module stream from /Users/nick/Desktop/TheOnlineTax_RnNuocity/node_modules/browserify-sign/browser/index.js: Module does not exist in the module map
  • 报错 Unknown argument type ‘attribute‘ in method -[RCTAppState getCurrentAppState:error:]. Extend RCTConvert to support this type.
  • RN网络请求不了 https ？

打包步骤

  1. 更改Config.js 线上地址为 http://www.nuocity.com/xnw-wx-consumer/

  2. 检查CodePush环境（重要）更改index.js的通道deploymentKey:config.StagingDeployment 检查 StagingDeployment 和 ReleaseDeployment key 是否正确

  3. 在根目录新建文件夹relseae_ios 执行react-native bundle --entry-file index.js --platform ios --dev false --bundle-output relseae_ios/main.jsbundle --assets-dest relseae_ios命令

  4. 将relseae_ios中的main.jsbundle和asset 拖拽到ios项目根目录中 （copy选择真实文件夹 蓝色的）

  5. Appdelegate.m中切换debug和release渠道，打Adhoc或者release包

  6. 在项目根目录 执行 code-push release 项目名称 ./relseae_ios/ 版本号
    例：code-push release NuocityTest ./relseae_ios/ 2.1.5 -d Production -m true
    注：
    强制更新 –m true
    生产环境 –d Production

    查看更新情况：
    生成环境 code-push deployment history 项目名称 Production
    开发环境 code-push deployment history 项目名称 Staging
    或者 code-push deployment ls 项目名称
